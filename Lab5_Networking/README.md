<h1><a id="user-content-part-1---installation-and-running-xv6" class="anchor" aria-hidden="true" href="#part-1---installation-and-running-xv6"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Part 1 - Installation and running xv6</h1>

<h1><a id="user-content-part-2---where-do-we-begin-grep-and-exploring" class="anchor" aria-hidden="true" href="#part-2---where-do-we-begin-grep-and-exploring"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Part 2 - Where do we begin? grep and exploring</h1>
<h2><a id="user-content-task-1---greping-around-to-learn-part-1" class="anchor" aria-hidden="true" href="#task-1---greping-around-to-learn-part-1"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Task 1 - Grep'ing around to learn part 1</h2>
<p>Remember the <code>grep</code> command? (<code>man grep</code>) It is a handy way to search text, and also a great way to navigate large projects. So if you have a large project like xv6, where would you first start?</p>
<ol>
<li>Step 1 -- find the main function.</li>
<li>If I was grep'ping around the source I would likely start with something like: <code>grep -ir -e "main(" .</code></li>
</ol>
<ul>
<li>'i' means case-insensitive. 'r' means do a recursive search 'e' is the pattern 'main(' which I am looking for, and the '.' at the end tells me where to start this search.</li>
<li>After this investigation, I find <code>main.c</code> which contains the main function. Starting from main.c is a good way to start understanding a programs <a href="https://en.wikipedia.org/wiki/Control_flow" rel="nofollow">control flow</a>.</li>
</ul>
<h2><a id="user-content-getting-familiar-with-xv6" class="anchor" aria-hidden="true" href="#getting-familiar-with-xv6"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Getting familiar with xv6.</h2>
<ol>
<li>Launch qemu and begin to run xv6 for your respective operating system.</li>
<li>While running the emulator type <code>ls</code> and you will see all of the commands you have available.
<ul>
<li>(While xv6 does not have as many as we have available in Unix--it is a start!)</li>
</ul>
</li>
</ol>
<p><strong>Discuss with your partner:</strong> <em>List five commands you observe with your partner that look familiar from Unix</em></p>
<ol>
<li>ls</li>
<li>cat</li>
<li>grep</li>
<li>mkdir</li>

<h1><a id="user-content-part-5---implementing-a-system-call-extending-xv6-further" class="anchor" aria-hidden="true" href="#part-5---implementing-a-system-call-extending-xv6-further"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Part 5 - Implementing a System Call (Extending xv6 further).</h1>
<p>(Reference Chapter 3 for more: <a href="https://pdos.csail.mit.edu/6.828/2014/xv6/book-rev8.pdf" rel="nofollow">https://pdos.csail.mit.edu/6.828/2014/xv6/book-rev8.pdf</a>)</p>
<p>We have seen conceptually how system calls worked, and even used them in assembly(e.g. write) and our C code (e.g. brk). When we use these system calls, our user programs are requesting the kernel to perform some action. In order for the kernel to perform this action, a <a href="https://en.wikipedia.org/wiki/Context_switch" rel="nofollow">context switch</a> occurs, the action is performed, and then the user program gets control again.</p>
<p>Recall at a high level what happens during a system call.</p>
<ol>
<li>A trap is issued. This stops the process from running.</li>
<li>The process needs its registers saved.</li>
<li>We switch to kernel mode and perform work.</li>
<li>Restore our registers with any applied changes as we give the user program back control.</li>
</ol>
